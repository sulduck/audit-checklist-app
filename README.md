# Audit Checklist App

감사 업무에서 사용할 수 있는 간단한 체크리스트 웹앱입니다. 실제 회사명, 개인정보, 내부자료 없이 예시 수준의 점검 항목을 사용해 테스트할 수 있습니다.

## 실행 방법

별도 설치나 빌드 과정이 필요하지 않습니다.

1. 저장소를 내려받습니다.
2. `index.html` 파일을 브라우저에서 엽니다.
3. 점검 항목, 분야, 우선순위를 입력한 뒤 **추가** 버튼을 눌러 체크리스트를 관리합니다.
4. 목록 상단의 **검색** 입력창에 항목명, 분야, 우선순위 또는 완료 여부를 입력해 필요한 항목만 확인합니다.

로컬 서버로 실행하려면 다음 명령을 사용할 수도 있습니다.

```bash
python3 -m http.server 8000
```

그다음 브라우저에서 `http://localhost:8000`에 접속합니다.

## 자동 테스트 실행 방법

검색, 완료 처리, 삭제, 통계 계산, 기존 데이터 기본값 보정 로직은 Node.js 기본 기능만 사용해 테스트할 수 있습니다.

```bash
npm test
```

테스트는 외부 테스트 라이브러리 없이 `node:test` 대신 `node:assert/strict`와 간단한 자체 실행기를 사용합니다.

## 주요 기능

- 점검 항목 추가
- 분야 선택: 계약, 예산, 복무, 보안, 기타
- 우선순위 선택: 높음, 보통, 낮음
- 목록에서 항목명, 분야, 우선순위, 완료 여부를 한눈에 확인
- 항목명, 분야, 우선순위, 완료 여부 기준 검색
- 전체 항목 수와 완료 항목 수 상단 표시
- 항목 완료 표시 및 완료 취소
- 항목 삭제
- 브라우저 `localStorage`를 사용한 새로고침 후 목록 유지
- 새로 추가된 분야와 우선순위 정보도 `localStorage`에 함께 저장
- 외부 라이브러리 없이 HTML, CSS, JavaScript만 사용
- 공공기관 업무용 화면을 참고한 단정하고 읽기 쉬운 PC 중심 디자인
- 항목 완료 표시 및 완료 취소
- 항목 삭제
- 브라우저 `localStorage`를 사용한 새로고침 후 목록 유지
- 외부 라이브러리 없이 HTML, CSS, JavaScript만 사용
- 공공기관 업무용 화면을 참고한 단정하고 읽기 쉬운 디자인

## 수동 테스트 방법

브라우저 스크린샷 검증 없이 다음 순서로 검색 기능을 확인할 수 있습니다.

1. `python3 -m http.server 8000` 명령으로 로컬 서버를 실행합니다.
2. 브라우저에서 `http://localhost:8000`에 접속합니다.
3. 서로 다른 항목명, 분야, 우선순위를 가진 점검 항목을 3개 이상 추가합니다.
4. 검색 입력창에 항목명의 일부를 입력해 해당 항목만 표시되는지 확인합니다.
5. 검색 입력창에 `계약`, `높음`, `완료`처럼 분야, 우선순위, 완료 여부 값을 입력해 필터링되는지 확인합니다.
6. 검색 결과가 없을 만한 단어를 입력해 빈 결과 안내 문구가 표시되는지 확인합니다.
7. **검색 초기화** 버튼을 눌러 전체 목록이 다시 표시되는지 확인합니다.
8. 검색 중 항목 완료, 완료 취소, 삭제 기능이 기존처럼 동작하는지 확인합니다.
